### How It Works

**1. LLM**

It reads your prompt and predicts the most likely next words, one token at a time. No memory between sessions. No goals. Just very good next-token prediction.

**2. Agent**

An agent is an LLM wrapped in a loop. It can decide what action to take, use tools, observe the result, and continue until the task is done.

```mermaid
flowchart LR
    G[Goal / Prompt] --> LLM[LLM]
    LLM --> A[Action]
    A --> T[Tool]
    T --> E[Environment]
    E --> O[Observation]
    O --> LLM
```

An agent is not just "answering" — it is thinking, acting, observing, and adjusting.

**3. Coding Agent**

A coding agent is an agent specialized for software work. Claude Code uses tools like `bash`, `read`, `edit`, `grep`, and `glob` to navigate and modify a codebase autonomously.

Real examples from this project:
- **Bug fix**: "the login button does nothing" → agent greps for the click handler, reads the file, spots a missing `await`, edits the line, runs tests to confirm.
- **Add a feature**: "add a `/health` endpoint" → reads the router to understand the existing pattern, adds the route in the same style, runs the server, curls to verify `200 OK`.
- **Refactor**: "extract DB logic from `userController.js` into a service" → reads the controller, identifies all DB calls, creates `userService.js`, updates imports, runs tests.

### Claude Code Tool Categories

| Category | Tools | Purpose |
|---|---|---|
| Read | `Read`, `Glob`, `Grep` | Understand the codebase |
| Write | `Edit`, `Write` | Modify files |
| Execute | `Bash` | Run commands, tests, builds |
| Memory | `TodoWrite` | Track multi-step work |
| Delegate | `Agent` | Spawn sub-agents for parallel tasks |

---

### What Claude Code Is Not

- It does not have internet access by default (unless MCP tools are added).
- It does not remember previous conversations unless memory files are used.
- It does not push to git, deploy, or send messages without being asked (or explicitly permitted).
