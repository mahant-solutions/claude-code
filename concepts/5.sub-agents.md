## Sub-Agents

Sub-agents are specialized AI assistants that Claude can delegate tasks to. Each runs in its own context window with its own system prompt, tool access, and permissions.

---

### Why Sub-Agents

- **Parallel work** — run multiple agents concurrently instead of sequentially
- **Isolation** — keep a focused agent from polluting the main context
- **Specialization** — give an agent only the tools and permissions it needs
- **Model matching** — use Haiku for cheap search, Opus for complex reasoning

---

### Built-in Sub-Agents

Claude Code ships with a few built-in agents:

| Agent | Model | Purpose |
|---|---|---|
| `Explore` | Haiku | Fast, read-only codebase search |
| `Plan` | Sonnet | Context gathering during plan mode |
| `General-purpose` | Sonnet | Complex multi-step tasks |

---

### Creating a Custom Sub-Agent

Sub-agents are `.md` files with YAML frontmatter stored in `.claude/agents/` (project) or `~/.claude/agents/` (global).

```markdown
---
name: test-runner
description: Run tests and report failures. Use when asked to verify code or check if tests pass.
model: haiku
tools: Bash(uv run pytest:*)
---

Run the test suite and report results. Be concise — pass/fail count and failing test names only.
```

**Key frontmatter fields:**

| Field | Purpose |
|---|---|
| `name` | Identifier |
| `description` | When Claude should delegate to this agent |
| `model` | `haiku`, `sonnet`, `opus`, or a full model ID |
| `tools` | Allowlist of tools (omit to inherit all) |
| `disallowedTools` | Denylist |
| `maxTurns` | Cap on agentic turns |
| `background` | Run concurrently without blocking |

---

### Invocation

Three ways to trigger a sub-agent:

```
# 1. Natural language — Claude auto-delegates if description matches
"Run the tests and tell me what's failing"

# 2. @-mention — guaranteed delegation
@"test-runner (agent)" run the full suite

# 3. CLI flag — session-wide agent
claude --agent test-runner
```

---

### Foreground vs Background

- **Foreground** (default) — blocks until the agent finishes, can ask for permissions interactively
- **Background** (`background: true`) — runs concurrently, permissions must be pre-approved

Background agents are useful when you want to kick off multiple independent tasks at once and collect results later.

---

### Model Strategy

A practical pattern: use a cheap fast model for high-volume sub-tasks, reserve capable models for reasoning.

```
Main agent (Sonnet)
├── Explore agent (Haiku)  — search & read, fast & cheap
├── Explore agent (Haiku)  — runs in parallel
└── Plan agent (Opus)      — architecture decision, worth the cost
```
