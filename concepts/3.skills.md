## Skills

A skill is a reusable, named prompt that lives in a `SKILL.md` file. When you type `/skill-name`, Claude loads and executes that prompt with full context of the current conversation.

Skills let you encode multi-step workflows once and invoke them with a slash command — instead of retyping instructions every time.

---

### Where They Live

```
.claude/
└── <skill-name>/
    └── SKILL.md
```

Skills placed under the project's `.claude/` are project-scoped. Skills under `~/.claude/` are global.

---

### SKILL.md Structure

```markdown
---
name: skill-name
description: >
  One or more sentences describing when this skill should trigger.
  Be specific — Claude uses this to decide whether to auto-invoke the skill.
---

# Skill Title

## Step 1 — ...
## Step 2 — ...
```

The frontmatter `description` is the trigger condition. Write it to match the exact phrases and intent you'll use when you want the skill to fire.

---

### Example: `ollama-chat-test`

This project has a skill at `.claude/ollama-chat-test/SKILL.md`.

**Trigger description:**
> Run and write tests for the ollama-chat CLI project. Use this skill whenever the user asks to run tests, verify code, check quality, write new tests, or after finishing any feature implementation. Also triggers for "does this pass?", "check my code", "are tests green?", "coverage", or any request involving pytest, ruff, or pyrefly.

**What it does:**
1. Runs 4 checks in parallel: `pytest`, `ruff check`, `ruff format --check`, `pyrefly check`
2. Reports results in a structured order: tests → lint → format → types
3. On failure only: gives specific, actionable fixes with file:line references

This skill encodes the entire quality-check workflow. Without it, you'd have to tell Claude which commands to run and how to report results every single time.

---

### When to Create a Skill

Create a skill when:
- You run the same multi-step workflow repeatedly (test + lint + typecheck)
- The workflow has specific output format requirements
- You want Claude to auto-detect when to use it based on natural language
- The instructions are long enough that repeating them in chat is tedious

Don't create a skill for:
- One-off tasks
- Things simple enough to just ask directly
- Workflows that change frequently

---

### Invoking Skills

```
/ollama-chat-test       # explicit invocation
```

Claude also auto-invokes skills when your message matches the trigger description. If you say "are the tests green?" and a skill's description says it fires for that phrase, Claude will use it without you typing `/`.

---

### Skills vs CLAUDE.md

| | CLAUDE.md | Skills |
|---|---|---|
| Purpose | Persistent project context | Reusable named workflows |
| Invocation | Always loaded | On-demand or auto-triggered |
| Format | Free-form markdown | Frontmatter + step-by-step |
| Best for | Rules, standards, project overview | Multi-step procedures |
