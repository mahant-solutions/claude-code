## Commands

Claude Code has three kinds of commands: **built-in slash commands**, **custom commands** (`.md` files), and **skills** (which also get slash commands).

---

### Built-in Slash Commands

Provided by Claude Code out of the box.

| Command | What it does |
|---|---|
| `/help` | Show available commands and shortcuts |
| `/clear` | Clear conversation history |
| `/compact` | Summarize conversation to save context |
| `/commit` | Stage, write, and create a git commit |
| `/review-pr` | Review an open pull request |
| `/plan` | Enter plan mode — Claude proposes a plan before writing code |
| `/fast` | Toggle fast mode (same model, faster output) |
| `/memory` | View or manage memory files |
| `/cost` | Show token usage and cost for the current session |
| `/bug` | Report a Claude Code bug |

---

### Custom Commands

Custom commands are plain `.md` files that live in `.claude/commands/`. Each file becomes a slash command named after the file.

```
.claude/
└── commands/
    └── commit-push.md    →  /commit-push
```

**Example — `commit-push.md` from this project:**

```markdown
---
allowed-tools: Bash(git add:*), Bash(git status:*), Bash(git commit:*), Bash(git push:*)
description: Commit staged/unstaged changes and push to remote
---

## Context

- Current git status: !`git status`
- Current git diff (staged and unstaged changes): !`git diff HEAD`
- Current branch: !`git branch --show-current`
- Recent commits: !`git log --oneline -10`

## Your task

Based on the above changes:

1. Stage all relevant changes (avoid secrets like .env or credentials)
2. Create a single commit with an appropriate message that matches this repo's commit style
3. Push the current branch to origin
```

#### Key features of the command format

**`allowed-tools` frontmatter** — restricts which tools Claude can use while executing this command. This is a security scope: running `/commit-push` cannot call arbitrary bash, only the four listed git commands.

```
allowed-tools: Bash(git add:*), Bash(git status:*), Bash(git commit:*), Bash(git push:*)
```

**`!`shell`` injection** — backtick expressions prefixed with `!` are executed at invocation time and their output is inserted into the prompt as context. Claude sees live data, not placeholders.

```markdown
- Current git status: !`git status`
- Current branch: !`git branch --show-current`
```

This means when you run `/commit-push`, Claude already has the current diff, branch, and recent commits in context before it starts — without you having to paste anything.

---

### Commands vs Skills

These are two distinct mechanisms:

| | Commands (`.claude/commands/*.md`) | Skills (`.claude/<name>/SKILL.md`) |
|---|---|---|
| Invocation | Explicit only (`/name`) | Explicit or auto-triggered by natural language |
| Tool restriction | `allowed-tools` frontmatter | No restriction |
| Live context injection | `!`shell`` syntax | No |
| Best for | Tightly scoped, safe operations | Complex multi-step workflows |

Use a **command** when you want a focused, permission-scoped action (git ops, deploys).
Use a **skill** when you want a rich workflow that Claude can also trigger automatically.

### Hooks (Advanced)

Hooks are shell commands that fire automatically on tool lifecycle events — not invoked by you.

```json
// ~/.claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit",
        "hooks": [{ "type": "command", "command": "uv run ruff format $FILE" }]
      }
    ]
  }
}
```

Hook events: `PreToolUse`, `PostToolUse`, `Notification`, `Stop`

Hooks are useful for auto-formatting on save, blocking unsafe operations, or logging tool usage.
